## Decisions Made (headline)

- `app/portfolio/` sub-package, `routes.py` + `service.py` + `models.py`, pure functions on `(conn, cache, ...)`, Pydantic v2.
- 60s snapshot cadence wires via a NEW `register_tick_observer(callback)` method added to `MarketDataSource` ABC and BOTH implementations (`SimulatorDataSource`, `MassiveDataSource`). Observer is registered in `lifespan` after `source.start(tickers)`.
- `app.state.last_snapshot_at: float` holds the snapshot clock; trade-time snapshots reset it to avoid duplicates.
- Trade rejections use HTTP 400 with body `{"detail": {"error": <code>, "message": <human>}}`. Codes: `insufficient_cash`, `insufficient_shares`, `unknown_ticker`, `price_unavailable`. 422 stays reserved for pydantic schema failures.
- Service raises domain exceptions (`InsufficientCash`, `InsufficientShares`, `UnknownTicker`, `PriceUnavailable`, base `TradeValidationError`); the route translates to `HTTPException`. Phase 5 chat auto-exec will catch these directly — that is why the service must not raise HTTP types.
- Reject trade when `cache.get_price(ticker) is None` (no silent fallback to seed prices).
- Trade ticker must be in the `watchlist` table (watchlist set = cache set, PLAN.md §6).
- DELETE the `positions` row when a sell zeros quantity (epsilon `abs(new_qty) < 1e-9`).
- BUY adding to existing position uses weighted average cost: `(old_qty*old_avg + buy_qty*buy_price) / (old_qty + buy_qty)`.

## Critical Anti-Patterns (do NOT repeat these)

- Do NOT spawn a separate `asyncio.Task` for the 60s snapshot loop — PLAN.md §7 explicitly says "piggybacked on the existing price-update loop (no separate background task)". The tick-observer callback is the sanctioned path.
- Do NOT raise `HTTPException` from `app/portfolio/service.py`. Phase 5 chat imports the service and needs to catch domain exceptions, not HTTP types.
- Do NOT fall back to `SEED_PRICES[ticker]` when the cache has no price — reject with `price_unavailable`.
- Do NOT allow trades on tickers absent from the `watchlist` table — reject with `unknown_ticker`. Auto-adding to watchlist is explicitly rejected (CONTEXT.md D-14).
- Do NOT keep `positions` rows with `quantity == 0`. Delete when zeroed (epsilon-based check for float residuals on fractional sells).
- Do NOT use `Decimal` for trade math — floats are adequate for the simulated $10k demo (Claude's Discretion, 03-CONTEXT.md).
