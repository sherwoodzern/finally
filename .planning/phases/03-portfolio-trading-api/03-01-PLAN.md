<objective>
Extend `MarketDataSource` with a zero-arg `register_tick_observer(callback)` hook and implement it in both `SimulatorDataSource` and `MassiveDataSource`. Callbacks fire on the asyncio event loop thread after cache writes, wrapped in narrow try/except per-callback so a broken observer does not kill the tick/poll loop (D-04, D-08).

Purpose: Provides the producer-side primitive that Plan 03-03 uses to piggyback a 60-second snapshot writer on the existing price-update loop (PLAN.md §7 "no separate background task"). PORT-05 foundation.

Output: ABC extension (one new `@abstractmethod`), two concrete impls (one new attribute + one new method + one nested-try observer loop each), plus a new test module that proves the contract and exception isolation.

From backend/app/market/simulator.py (current `__init__` and `_run_loop`):
```python
class SimulatorDataSource(MarketDataSource):
    def __init__(self, price_cache, update_interval=0.5, event_probability=0.001):
        self._cache = price_cache
        self._interval = update_interval
        self._event_prob = event_probability
        self._sim: GBMSimulator | None = None
        self._task: asyncio.Task | None = None

    async def _run_loop(self) -> None:
        """Core loop: step the simulation, write to cache, sleep."""
        while True:
            try:
                if self._sim:
                    prices = self._sim.step()
                    for ticker, price in prices.items():
                        self._cache.update(ticker=ticker, price=price)
            except Exception:
                logger.exception("Simulator step failed")
            await asyncio.sleep(self._interval)
```

From backend/app/market/massive_client.py (current `__init__` and critical section of `_poll_once`):
```python
class MassiveDataSource(MarketDataSource):
    def __init__(self, api_key, price_cache, poll_interval=15.0):
        self._api_key = api_key
        self._cache = price_cache
        self._interval = poll_interval
        self._tickers: list[str] = []
        self._task: asyncio.Task | None = None
        self._client: RESTClient | None = None

    async def _poll_once(self) -> None:
        if not self._tickers or not self._client:
            return
        try:
            snapshots = await asyncio.to_thread(self._fetch_snapshots)
            processed = 0
            for snap in snapshots:
                try:
                    price = snap.last_trade.price
                    timestamp = snap.last_trade.timestamp / 1000.0
                    self._cache.update(ticker=snap.ticker, price=price, timestamp=timestamp)
                    processed += 1
                except (AttributeError, TypeError) as e:
                    logger.warning("Skipping snapshot for %s: %s", ..., e)
            logger.debug("Massive poll: updated %d/%d tickers", processed, len(self._tickers))
        except Exception as e:
            logger.error("Massive poll failed: %s", e)
```

Concurrency contract (from 03-RESEARCH.md Pitfall 2):
- Observers MUST fire on the asyncio event loop thread, not inside `asyncio.to_thread`.
- In `_poll_once`, observer invocation is after `await asyncio.to_thread(...)` returns — already on the loop.
- In `_run_loop`, everything happens on the loop thread.
