Create `.env.example` at the repo root with the EXACT content below (verbatim from CONTEXT.md D-12). Three keys, in order: required first (`OPENROUTER_API_KEY`), optional second (`MASSIVE_API_KEY`), mock-mode last (`LLM_MOCK`). Each key has a comment block explaining what happens when the value is the default.

```
# FinAlly environment configuration
# Copy this file to .env and edit the values you need.

# Required for AI chat. Get one at https://openrouter.ai/.
# If left empty, /api/chat returns 502 and the chat panel surfaces an error,
# but the rest of the terminal (heatmap, P&L, watchlist, trades) still works.
OPENROUTER_API_KEY=

# Optional. If set and non-empty, switches to the real Massive (Polygon.io)
# market data feed. Otherwise the simulator runs (recommended for the demo).
MASSIVE_API_KEY=

# Set to "true" to make the LLM return deterministic mock responses (useful
# for E2E tests and local development without an API key).
LLM_MOCK=false
```

Format constraints (PATTERNS.md):
- `KEY=value` form (no spaces around `=`). `--env-file` is finicky on spaces.
- No quotes around values.
- Newline-terminated final line.
- No emojis (CLAUDE.md).
- DB_PATH is intentionally NOT exposed (container-internal; set by Dockerfile to `/app/db/finally.db`).

Critical: Do NOT include any other env vars (no `DB_PATH`, no `PORT`, no `LOG_LEVEL`). The file is the contract surface for end users; PLAN.md §5 lists exactly these three.
