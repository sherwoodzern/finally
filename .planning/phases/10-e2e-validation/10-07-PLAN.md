    1. **Add a helper at the top of the test body** (or inline; helper is fine since the spec is small) that performs the dismissal. Use BOTH techniques — Escape key dispatch AND mouse displacement — because the verifier's evidence (`<td class="px-4 font-semibold">META</td> ... subtree intercepts pointer events`) shows the tooltip element actually intercepts pointer events, so simply moving the mouse is insufficient when the tooltip has already pinned itself to the cell.        Place this helper inside the test function, immediately after `await page.goto('/');`:

       ```typescript
         // Dismiss any lingering Recharts hover tooltip before clicking a sibling
         // tab. Pointer-event interception by the tooltip <td>{ticker}</td>
         // subtree is the documented failure mode (10-VERIFICATION.md Gap Group B,
         // all 3 browsers). The previous `page.mouse.move(0, 0)`-only mitigation
         // was insufficient because once the tooltip pins to a cell, moving the
         // cursor away does not retract the overlay; Escape is what tells the
         // Recharts tooltip lifecycle to dismiss. Both steps together (Escape +
         // mouse displacement) are robust across chromium, firefox, and webkit.
         const dismissChartTooltip = async () => {
           await page.keyboard.press('Escape');
           await page.mouse.move(0, 0);
         };
       ```

       b. Immediately BEFORE the `await page.getByTestId('tab-pnl').click();` line (currently line 41), call:
