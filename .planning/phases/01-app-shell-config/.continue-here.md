## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| ASGITransport drains SSE | `httpx.ASGITransport` buffers the full ASGI response and cannot drain an infinite `while True` SSE generator. Plan 01-03 initially prescribed this and tests hung on 5s ReadTimeouts. | advisory | For streaming tests, spawn a real in-process uvicorn server bound to `127.0.0.1:<random_port>` and use a regular `httpx.AsyncClient`. ASGITransport is fine only for finite responses. |
| Module-level APIRouter accumulates routes | `create_stream_router` originally used a module-level `APIRouter`; the factory re-registered `/prices` on every call, and Starlette dispatched the *first* match — a stale closure over a dead `PriceCache`. | advisory | Router factories must construct a fresh `APIRouter()` inside the function body. Fixed in commit `704525d`. Do not reintroduce module-level router state in stream.py / future routers. |
| Lifespan include_router never unwound | `lifespan.py:44` runs `app.include_router(create_stream_router(cache))` inside startup but nothing removes it on shutdown. Re-entering the lifespan on the same `app` instance stacks duplicate routes closing over a dead cache (WR-01 in 01-REVIEW.md). | advisory | Production lifespan runs once so this is latent. If a later phase needs lifespan re-entry (graceful reload, test reuse), move the `include_router` call to `main.py` at app-build time. Each new app instance avoids the issue. |
