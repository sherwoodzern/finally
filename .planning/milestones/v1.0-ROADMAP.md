# Milestone v1.0: MVP — AI Trading Workstation

**Status:** SHIPPED 2026-04-28
**Phases:** 1-11
**Total Plans:** 51 (all complete)

## Overview

FinAlly v1.0 ships a single-Docker-command Bloomberg-style AI trading workstation: live SSE price streaming for 10 default tickers, a simulated $10k portfolio with instant-fill market orders, a Bloomberg-terminal three-column layout (watchlist · workspace · positions) plus a collapsible AI chat drawer, and an LLM copilot (LiteLLM → OpenRouter → Cerebras `gpt-oss-120b`) that auto-executes trades and watchlist changes via structured outputs. Static Next.js export served by FastAPI on `:8000`; SQLite persisted in a Docker volume; Playwright canonical harness exits 0 with `21 passed / 0 failed / 0 flaky` reproducibly across two consecutive runs and three browsers.

## Milestone Summary

**Decimal Phases:** None (no decimal phases inserted into v1.0).

**Key Decisions:**

- **Single Docker container, single port (8000)** — students run one command; no docker-compose for production.
- **FastAPI serves Next.js static export from `/`** — same-origin, no CORS, one process for `/api/*` and the static frontend.
- **SSE over WebSockets** — one-way server→client push is all the price stream needs; native `EventSource` reconnect.
- **SQLite + lazy schema init in FastAPI lifespan** — no auth → no multi-user → no need for a DB server.
- **Market-orders-only portfolio** — eliminates order book, limit-order logic, partial fills.
- **Two market-data sources behind one ABC** — `SimulatorDataSource` (correlated GBM) is default; `MassiveDataSource` activates only when `MASSIVE_API_KEY` is set.
- **LiteLLM → OpenRouter to `openrouter/openai/gpt-oss-120b` with Cerebras inference** — fast enough that token streaming is not needed; structured outputs drive auto-execution.
- **Auto-executed trades from chat (no confirmation dialog)** — stakes are zero (fake money) and the agentic-AI moment is the demo.
- **`LLM_MOCK=true` deterministic chat path** — free, fast, reproducible E2E tests; key-less local dev.
- **Zustand for the live price store + TanStack Query for portfolio/history** — push-driven prices need a global, mutation-light store; pull-driven REST resources benefit from cache + refetchInterval.
- **Lightweight Charts for ticker chart + sparklines, Recharts for heatmap + P&L** — canvas vs SVG split per workload characteristics.
- **Tab-based center column (Chart / Heatmap / P&L)** — density: every pixel earns its place.
- **Per-project Playwright viewport 1440×900** — aligns the test environment with the design contract; avoids spurious layout-overlap failures at the device default 1280×720.
- **`tmpfs:/app/db` on the test-side appsvc** — gives the canonical command's `reproducibly` clause teeth without changing the command, without a `down -v` wrapper, and without a test-only reset endpoint.
- **Option B for `human_needed` closure** — preserve enum, add `human_acceptance: indefinite` sibling key with dated rationale (Phase 11 plan 11-03 G3+G4 closure).
